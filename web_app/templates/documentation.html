What is AI IDS?

The AI Intrusion Detection System is a machine learning-powered network security solution that monitors network traffic in real-time to detect potential threats and attacks.

Monitor network packets as they flow through your system with millisecond-level latency.

üåê Web Dashboard

üîç Supported Attack Types

The system can detect over 20 different types of network attacks including:

  • DoS/DDoS Attacks: neptune, back, land, pod, smurf, teardrop
  • Reconnaissance: ipsweep, nmap, portsweep, satan
  • Unauthorized Access: ftp_write, guess_passwd, imap, multihop
  • Data Attacks: buffer_overflow, loadmodule, perl, rootkit
  • And many more...

Windows

  • Run Command Prompt as Administrator for packet capture

macOS

  • Install Homebrew if not already installed
  • Use sudo for packet capture functionality
  • Allow network access when prompted

Linux

  • Install libpcap development headers
  • Use sudo for raw socket access
  • Consider running as a service for production

‚öôÔ∏è Configuration

Configuration File

The system uses a YAML configuration file located at src/ids_config.yaml:

network: interface: 'auto' # Network interface to monitor buffer_size: 1000 # Packet buffer size promiscuous_mode: false detection: model_path: 'models/random_forest_model_tuned.joblib' confidence_threshold: 0.8 # Alert threshold (0.0-1.0) rate_limit_window: 60 # Rate limiting window (seconds) rate_limit_max: 5 # Max alerts per window per IP monitoring: enable_web_dashboard: true dashboard_port: 8080 dashboard_host: '127.0.0.1' stats_update_interval: 5 alerts: log_file: 'security_alerts.json' enable_email: false email_smtp: 'smtp.gmail.com' email_port: 587 email_user: '' email_password: '' email_recipients: [] logging: level: 'INFO' file: 'intrusion_detection.log' max_size_mb: 10 backup_count: 5

üîß Key Configuration Options

Network Settings

  • interface: Network interface to monitor (auto-detect or specify)
  • buffer_size: Number of packets to buffer before processing
  • promiscuous_mode: Enable promiscuous mode for interface

Detection Settings

  • confidence_threshold: Minimum confidence for alerts (0.0-1.0)
  • rate_limit_window: Time window for rate limiting alerts
  • rate_limit_max: Maximum alerts per IP in time window

Dashboard Settings

  • dashboard_port: Port for web dashboard (default: 8080)
  • dashboard_host: Host binding for dashboard (default: 127.0.0.1)
  • stats_update_interval: How often to update statistics
